离线部署kubernetes ingress nginx controller：v0.48.1@TOC

1.因是内网环境，需要离线安装，先在外网下载好镜像。

首先在github看下官方最新版本：https://github.com/kubernetes/ingress-nginx/releases

在github下载这个版本的deploy.yaml
https://github.com/kubernetes/ingress-nginx/blob/controller-v0.48.1/deploy/static/provider/baremetal/deploy.yaml
看看需要下载几个镜像，
看到需要下载两个：k8s.gcr.io/ingress-nginx/controller:v0.48.1，docker.io/jettech/kube-webhook-certgen:v1.5.1
docker pull k8s.gcr.io/ingress-nginx/controller:v0.48.1
网络是不通的。
所以我们找一下国内的阿里源，找到这个是可以用的，把版本号替换成v0.48.1

docker pull registry.aliyuncs.com/google_containers/nginx-ingress-controller:v0.48.1
docker pull docker.io/jettech/kube-webhook-certgen:v1.5.1

  • 2

改回官方名

docker tag  registry.aliyuncs.com/google_containers/nginx-ingress-controller:v0.48.1 k8s.gcr.io/ingress-nginx/controller:v0.48.1

打tar包

docker save -o ingress-nginx-controller-v0.48.1.tar k8s.gcr.io/ingress-nginx/controller:v0.48.1
docker save -o kube-webhook-certgen-v1.5.1.tar docker.io/jettech/kube-webhook-certgen:v1.5.1

然后把tar包拷贝的内网环境，上传到内网的镜像仓库

docker load -i  ingress-nginx-controller-v0.48.1.tar
docker load -i kube-webhook-certgen
docker tag  k8s.gcr.io/ingress-nginx/controller:v0.48.1 10.23.120.66/public/ingress-nginx-controller:v0.48.1
docker tag jettech/kube-webhook-certgen:v1.5.1 10.23.120.66/public/kube-webhook-certgen:v1.5.1
docker push 10.23.120.66/public/ingress-nginx-controller:v0.48.1
docker push 10.23.120.66/public/kube-webhook-certgen:v1.5.1

2.部署ingress-nginx-controller，相当于部署一个nginx

在github下载这个版本的deploy.yaml
https://github.com/kubernetes/ingress-nginx/blob/controller-v0.48.1/deploy/static/provider/baremetal/deploy.yaml
把image改为自己的镜像
k8s.gcr.io/ingress-nginx/controller:v0.48.1
改为10.23.120.66/public/ingress-nginx-controller:v0.48.1

kubectl apply -f deploy.yaml

看看pod是否正常启动

kubectl get pods -n ingress-nginx

  • 1

kubectl get svc -n ingress-nginx

  • 1

可以看到是nodePort方式，这个9664端口相当于是nginx的入口

3.增加ingress配置，相当于增加nginx的转发配置

---
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: ingress-test
  annotations:
    nginx.ingress.kubernetes.io/use-regex: "true"
spec:
  rules:
  - http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: test_service
            port:
              number: 8086
